コード書きだして一年未満のゴミが書いたプログラムなので、

知識でどうにかなることではなく、論理的思考ができなければ実装できないプログラムを作成したほうがいいと
先生に言われたため作成に至った

VSプレイヤーの思考ルーチンは再帰と評価値・負けそうな手を防ぐのを重視して、
モンテカルロ法でとりあえずAI探索のアルゴリズムを実装。
置いて負けにならないで置くことができる手（合法手と呼ぶ）におく→
そのあとはランダムで置いて（負けない手に置く判定は入れている）
いき各合法手を難易度に応じて回数分プレイアウトしている。
試行回数を多くしても評価関数が微妙なのであんまり意味ないぽい。

コードはマジックナンバー多々あり。
はじめはきれいに作れていたが、設計がきちんとできていなかったので、
コードを追加していき、徐々に変なコードになってしまった。
理想は関数をもっと分割して30～40位で作ればよかった。
